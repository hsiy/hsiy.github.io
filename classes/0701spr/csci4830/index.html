Description

The course focuses on the engineering and development of real-world software products. It provides a pragmatic introduction into the fundamentals of software engineering. Basics of software project management are presented and are practiced in the term project.

Learning Objectives

Students who complete this course will:
  • have knowledge of process, methods, tools used in the software engineering profession,
  • be familiar with concepts of software engineering project management,
  • be able to apply the lessons learned to real-world software projects.

Course content:

  1. Overview and motivations
  2. Requirements analysis
  3. Software design
  4. Verification and Validation
  5. Software evolution
  6. Project management
  7. Advanced topics

Term Project

Students will work together in teams of 3-4 on a term project to be prepared by the instructor. The project will require a significant time commitment from the beginning to the end of the semester.

Term project objectives:

  • Experience practical issues of project management.
    • Learn to use a project management tool.
    • Learn to plan ahead and modify the plan as needed.
  • Task distribution, teamwork and coordination.
    • Learn to break down tasks and define responsibilities.
  • Apply problem definition techniques.
    • Learn to elicit requirements from user.
    • Learn to use object-oriented analysis techniques.
  • Create solution systematically.
    • Learn to use object-oriented design techniques.
    • Develop implementation that is consistent with design.
    • Employ a version control system.
  • Gain experience with validation techniques.
    • Learn to conduct formal reviews and inspections.
    • Learn to plan tests early in the process.
    • Learn to use simple automated testing tools.
